# 基于云的协同机械 CAD 系统实施前准备清单与解决方案

## 1. 文档说明

本文档说明在正式编码实现“基于云的协同机械 CAD 系统”之前，需要完成哪些准备工作，以及每项准备工作的具体解决方案。

根据当前团队选择，本项目采用“Vue 前端 + Spring Boot 后端”的主技术路线。本文档已经按该路线更新，不再以 React、NestJS、Prisma 作为主方案。

## 2. 最终推荐技术路线

### 2.1 前端技术栈

```text
Vue 3
TypeScript
Vite
Vue Router
Pinia
Three.js
three-bvh-csg
Element Plus
Axios
@stomp/stompjs
```

作用：

1. Vue 3：构建 CAD 工作台界面。
2. TypeScript：约束 CAD 数据结构、协同消息、API 响应类型。
3. Vite：提供前端开发服务器和生产构建。
4. Vue Router：管理登录页、项目列表页、CAD 工作台页。
5. Pinia：管理登录态、项目态、CAD 文档态、协同连接态。
6. Three.js：渲染 3D 场景、草图、拉伸体、辅助网格和相机控制。
7. three-bvh-csg：在基础封闭 mesh 上实现课程级 Cut、布尔并集和差集。
8. Element Plus：提供表单、弹窗、表格、消息提示等通用管理界面组件。
9. Axios：调用后端 REST API。
10. `@stomp/stompjs`：连接 Spring WebSocket STOMP 服务，实现协同消息订阅和发送。

### 2.2 后端技术栈

```text
Java 21
Spring Boot
Spring Web MVC
Spring Security
JWT
Spring WebSocket + STOMP
Spring Data JPA
Hibernate
PostgreSQL
Flyway
Redis，可选
Lombok，可选
Spring Boot Actuator，可选
```

作用：

1. Java 21：后端运行语言，选择 LTS 版本。
2. Spring Boot：后端主框架，负责工程启动、依赖管理和自动配置。
3. Spring Web MVC：提供 REST API。
4. Spring Security + JWT：实现登录鉴权、接口保护和 WebSocket 鉴权。
5. Spring WebSocket + STOMP：实现协同房间、主题订阅、操作广播。
6. Spring Data JPA + Hibernate：实现用户、项目、文档、版本等实体持久化。
7. PostgreSQL：保存业务数据、模型 JSON、版本快照。
8. Flyway：管理数据库表结构迁移。
9. Redis：后续用于在线用户、房间状态、协同锁、消息扩展。
10. Actuator：后续用于健康检查和部署监控。

### 2.3 部署与云服务

```text
华为云 ECS
Nginx
Docker Compose
PostgreSQL 容器或华为云 RDS
Redis 容器或华为云 DCS
本地文件存储或华为云 OBS
```

MVP 建议：

1. ECS 单机部署。
2. Nginx 统一代理前端静态资源、后端 API、WebSocket。
3. PostgreSQL 先用 Docker 容器。
4. Redis 先可选。
5. 模型快照先存 PostgreSQL JSONB。
6. OBS 作为增强项接入。

### 3.4 分支策略

```text

fix/*     修复分支

```

规则：

1. `main` 始终保持可演示。
2. 所有开发先合并到 `dev`。

## 7. 数据库准备

### 7.1 本地 PostgreSQL

建议用 Docker Compose：

```yaml
services:
  postgres:
    image: postgres:16
    container_name: cloud-cad-postgres
    environment:
      POSTGRES_USER: cad_user
      POSTGRES_PASSWORD: cad_password
      POSTGRES_DB: cloud_cad
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:
```

启动：

```powershell
docker compose up -d postgres
```

### 7.2 推荐数据库表

首批表：

1. `users`
2. `projects`
3. `project_members`
4. `documents`
5. `document_versions`
6. `operation_logs`

### 7.3 Flyway 迁移文件

建议首个迁移文件：

```text
backend/src/main/resources/db/migration/V1__init_schema.sql
```

建议由 Flyway 管理表结构，不要依赖 Hibernate 自动建表作为正式方案。

### 7.4 模型 JSON 存储策略

MVP 阶段：

1. `document_versions.snapshot_json` 使用 PostgreSQL `jsonb` 字段。
2. 当前文档只保存 `current_version_id`。
3. 每次保存新增版本，不覆盖旧版本。

增强阶段：

1. 大模型快照存 OBS。
2. 数据库保存 `snapshot_path`。
3. 后端 `StorageService` 屏蔽本地文件和 OBS 的差异。

## 8. Redis 准备

### 8.1 是否必须使用 Redis

MVP 不强制使用 Redis。

可以先把在线用户和协同房间保存在 Spring Boot 单实例内存中。

### 8.2 什么时候接入 Redis

建议在以下情况接入：

1. 需要保存短期协同锁。
2. 需要断线重连恢复在线状态。
3. 需要后端多实例部署。
4. 需要跨实例广播 WebSocket 消息。

### 8.3 本地 Redis

```yaml
services:
  redis:
    image: redis:7
    container_name: cloud-cad-redis
    ports:
      - "6379:6379"
```

## 9. WebSocket 与协同准备

### 9.1 推荐协议

由于后端选择 Spring Boot，协同协议推荐：

```text
STOMP over WebSocket
```

原因：

1. Spring 对 STOMP 支持成熟。
2. 可以用 topic 表达文档房间。
3. 前端订阅和发送语义清晰。
4. 比原生 WebSocket 更适合课程报告说明。

### 9.2 推荐端点设计

```text
WebSocket endpoint: /ws
订阅文档操作: /topic/documents/{documentId}/operations
订阅在线用户: /topic/documents/{documentId}/presence
发送操作: /app/documents/{documentId}/operations
发送光标: /app/documents/{documentId}/cursor
```

### 9.3 协同消息结构

```json
{
  "id": "msg-001",
  "type": "feature.added",
  "documentId": "doc-001",
  "userId": "user-001",
  "baseVersion": 3,
  "timestamp": 1710000000000,
  "payload": {}
}
```

### 9.4 准备完成标准

1. 前端可以连接 `/ws`。
2. 前端可以订阅 `/topic/documents/{id}/operations`。
3. 用户 A 发送操作，用户 B 可以收到。
4. WebSocket 连接需要携带 JWT 或建立连接后校验身份。

### 10.5 Nginx 代理准备

Nginx 需要代理：

1. 前端静态资源。
2. `/api/` 到 Spring Boot。
3. `/ws` 到 Spring WebSocket。

示例：

```nginx
server {
    listen 80;
    server_name _;

    root /var/www/cloud-cad/frontend;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080/api/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    location /ws {
        proxy_pass http://127.0.0.1:8080/ws;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}
```

## 11. OBS 文件存储准备

### 11.1 MVP 阶段策略

MVP 可以先不用 OBS。

原因：

1. 模型 JSON 数据量小。
2. PostgreSQL JSONB 足够保存版本快照。
3. 接入 OBS 会增加 AK/SK、权限、SDK、网络配置成本。

### 11.2 后续接入策略

后端定义统一接口：

```java
public interface StorageService {
    String putObject(String path, byte[] content);
    byte[] getObject(String path);
}
```

实现：

1. `LocalStorageService`：开发期使用。
2. `ObsStorageService`：部署增强期使用。

环境变量：

```text
OBS_ACCESS_KEY_ID=
OBS_SECRET_ACCESS_KEY=
OBS_BUCKET=
OBS_ENDPOINT=
```

注意：

1. 不要把 AK/SK 提交到 Git。
2. 不要在演示截图中暴露密钥。
3. OBS 桶建议设为私有。

## 17. 正式实现前的执行顺序

建议按以下顺序执行：

1. 初始化 Git 仓库。
2. 建立 `frontend/backend/docs/deploy/references` 目录。
3. 安装 Node、pnpm、JDK 21、Maven、Docker。
4. 创建 Vue 3 + Vite 前端项目。
5. 创建 Spring Boot 后端项目。
6. 用 Docker 启动 PostgreSQL。
7. 编写 Flyway 初始建表脚本。
8. 跑通后端健康检查接口。
9. 跑通前端到后端 API 请求。
10. 跑通 WebSocket STOMP 最小连接。
11. 准备华为云 ECS。
12. 开始业务功能开发。

## 18. 准备完成标准

正式写业务功能之前，应达到：

1. 所有人可以 clone 仓库。
2. 前端 `pnpm run dev` 可以启动。
3. 后端 `mvn spring-boot:run` 可以启动。
4. PostgreSQL 可以连接。
5. Flyway 可以执行迁移。
6. 前端能请求后端 `/api/health`。
7. 前端能连接后端 `/ws`。
8. Git 分支规范确定。
9. 华为云 ECS 创建方案确定。
10. 团队明确首版只做 MVP，不被高级功能拖住。

本文档用于核验当前项目设计是否满足实验设计 PDF 中“5. 功能需求说明”的最低要求，并记录本次为满足要求而做出的文档补充和功能优先级调整。

核验依据来自实验 PDF：

2. 5.2 后端功能最低要求。
3. `00附件-原始相关资料.txt` 中补充的 Web 三维建模要求：在浏览器中创建项目，并在项目内按参数创建球体、锥体、立方体/长方体，能够在浏览器中渲染显示并支持旋转、拉伸/尺寸调整和拖拽。

### 2.4 参数化三维实体创建

新增原始资料要求：

1. 用户可以在浏览器中创建一个项目。
2. 在项目内通过参数创建球体，至少支持输入半径。
3. 在项目内通过参数创建锥体，至少支持输入底面参数和高度；课程实现中采用底面半径 + 高度。
4. 在项目内通过参数创建立方体/长方体，至少支持输入长、宽、高。
5. 创建后的实体能在浏览器中渲染显示，并支持旋转视图、尺寸调整和拖拽移动。

当前设计：

1. 项目创建、项目列表、进入工作台已列为 P0，并由后端项目接口和默认文档支持。
2. `CadDocument.features` 新增 `box`、`sphere`、`cone` 三类参数化特征。
3. 工具栏新增“立方体”“球体”“锥体”入口，通过弹窗输入长宽高、半径、底面半径和高度。
4. Three.js 几何内核通过 `BoxGeometry`、`SphereGeometry`、`ConeGeometry` 生成可渲染 mesh。
5. 右侧属性面板支持精确修改三维实体坐标和尺寸参数。
6. 视口通过 Raycaster 拾取三维实体，支持拖拽移动参数化实体。
7. 视图旋转由 OrbitControls 支持，鼠标右键旋转、中键平移、滚轮缩放。
8. 参数化实体的创建和更新通过 `feature.created`、`feature.updated` 进入协同 operation 链路。

## 5. 简单 Cut/Boolean 的推荐实现方式

### 5.1 MVP 实现方式

推荐采用：

```text
Three.js 参数化几何 + three-bvh-csg
```

实现策略：

1. 拉伸矩形生成 Box mesh。
2. 拉伸圆生成 Cylinder mesh。
3. Cut 本质上转换为 Target Mesh - Cutter Mesh。
4. Boolean Union 转换为 Mesh A ∪ Mesh B。
5. Boolean Difference 转换为 Mesh A - Mesh B。
6. 特征树仍记录参数和引用关系，不只保存结果 mesh。

### 5.2 实现限制

首版限制：

1. 只支持基础封闭实体。
2. 不承诺复杂交叉拓扑稳定。
3. 不支持工业 CAD 的拓扑命名。
4. 不支持倒角、圆角、抽壳。

这样既满足实验要求，又能避免项目被工业几何内核复杂度拖住。

## 6. 低难度绘图增强设计

### 6.1 椭圆

数据结构建议：

```json
{
  "id": "ellipse-001",
  "type": "ellipse",
  "cx": 0,
  "cy": 0,
  "rx": 40,
  "ry": 20,
  "rotation": 0
}
```

实现难度：低。

原因：

1. Three.js 或 Canvas/SVG 都容易画椭圆。
2. 不需要复杂约束求解。
3. 属性面板可直接编辑中心点、长轴和短轴。

### 6.2 多段线

数据结构建议：

```json
{
  "id": "polyline-001",
  "type": "polyline",
  "points": [
    { "x": 0, "y": 0 },
    { "x": 20, "y": 0 },
    { "x": 20, "y": 30 }
  ],
  "closed": false
}
```

实现难度：低到中。

原因：

1. 只需要连续点选。
2. 可用 Enter 或双击结束绘制。
3. 后续可扩展为封闭轮廓拉伸。

### 6.3 构造线

数据结构建议：

```json
{
  "id": "construction-001",
  "type": "line",
  "construction": true,
  "x1": 0,
  "y1": 0,
  "x2": 100,
  "y2": 0
}
```

实现难度：低。

原因：

1. 本质仍是线段。
2. 只需要用虚线或浅色显示。
3. 不参与拉伸轮廓生成。

### 6.4 网格吸附

实现策略：

1. 设置 gridSize，例如 5mm。
2. 鼠标坐标转换到草图坐标后，四舍五入到最近网格点。
3. 提供开关按钮。

实现难度：低。

### 6.5 基础尺寸标注

实现策略：

1. 矩形显示宽高。
2. 圆显示半径。
3. 线段显示长度。
4. 尺寸文本只作为草图辅助，不作为工程图模块。

实现难度：低。

## 8. 核验结论

更新后的文档已经满足实验 PDF 中 5.1、5.2 的最低要求，也覆盖了原始资料新增的“Web 中参数化创建球体、锥体、立方体/长方体并渲染、拖拽、调整”的要求。

1. 简单 Cut/Boolean 用轻量 CSG 达标。
2. 工业级几何内核后续增强。
3. 简单约束和尺寸驱动达标。
4. 通用约束求解器后续增强。
5. STOMP 操作广播达标。
6. Yjs/CRDT 后续增强。
7. 参数化三维基础体通过 Three.js 达标，后续可用 OpenCascade.js 提升工业几何精度。

因此，后续开发可以按当前文档进入工程实现阶段。

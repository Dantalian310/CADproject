# 13-CAD 功能增强与成熟产品调研记录

## 1. 本次问题背景

当前系统已经能够注册、登录、创建项目并进入工作台，但距离可用的在线 CAD 工具仍有明显差距，主要问题包括：

1. 项目列表只能创建和进入项目，缺少删除项目的入口。
2. 工作台内绘图能力较弱，圆、矩形等实体不能稳定创建在鼠标所在位置。
3. 草图实体创建后不能直接移动、调整尺寸或删除。
4. 工具栏存在 Undo/Redo 占位，但没有真正的撤销和重做能力。
5. 属性面板只展示文档信息，没有提供参数化编辑入口。

## 2. 成熟线上 CAD 产品启发

结合 Onshape、SketchUp for Web、Tinkercad、Fusion 等线上或云端 CAD 产品的共性，可以提炼出适合课程项目第一阶段落地的功能原则：

1. 草图实体必须可选中、可编辑、可删除。
2. 绘图应以鼠标位置映射到建模平面，而不是用屏幕像素粗略估算。
3. 基础图元应支持拖拽创建，并在拖拽过程中提供预览。
4. 可编辑参数应暴露到属性面板，例如圆心、半径、矩形位置、宽度、高度、线段端点。
5. 任何建模操作都应进入历史栈，至少支持撤销和重做。
6. 协同 CAD 中，本地创建、更新、删除都应能够通过操作事件同步到其他客户端。
7. 项目管理应具备完整生命周期，至少支持创建、查看、进入、成员管理和删除。
8. 工作区应具备常用视图导航能力，包括缩放、平移、旋转、切换标准视角和缩放到全图。
9. 常用编辑动作应同时提供按钮和快捷键，例如复制、删除、撤销、重做。
10. 草图绘制必须具备对象吸附能力，至少应覆盖端点、中点、圆心、角点等高频几何参考点。
11. 选择或绘制时应提供动态测量与定位反馈，使用户知道当前落点、尺寸和参考对象。

本轮补充核验了 Onshape 官方帮助文档：

1. Onshape Sketch Tools 文档列出线、角点矩形、中心点矩形、中心点圆、三点圆、圆弧、样条、点、构造线等草图工具，并说明草图工具用于创建生成 3D 特征的 2D 草图。对应本项目后续应继续扩展草图图元类型，但当前阶段优先补齐线、矩形、圆的高质量交互。
2. Onshape Sketch Tools 文档提到选择时会优先处理未完全约束的草图点，并支持在草图曲线和基准之间应用约束及尺寸。对应本项目，本轮先实现对象吸附和明确吸附反馈，后续再推进约束求解。
3. Onshape Measure Tool 文档强调选择实体时动态显示测量信息，并支持中心、半径、长度、点位置、实体间距离等测量类型。对应本项目，已实现动态尺寸反馈、选中尺寸标签，本轮继续补充吸附点坐标反馈。
4. Onshape Sketch Tools 文档说明草图支持复制、粘贴，并在粘贴后通过触摸拖拽或点击定位。对应本项目，本轮将复制能力扩展为旋转、镜像复制和矩形阵列复制，使基础草图编辑更接近机械 CAD 的重复建模工作流。
5. Onshape Sketch Tools 文档明确草图曲线可以应用约束和尺寸。对应本项目，本轮将原先的固定、水平、垂直、尺寸按钮从“只记录约束数据”升级为“实际驱动实体几何”的功能闭环。
6. Onshape Measure Tool 文档强调两个实体间可显示中心距离、最小/最大距离等关系测量，Onshape Sketch Tools 文档也说明草图曲线可应用约束。对应本项目，本轮继续补充实体之间的同心、等半径和平行关系约束。

## 3. 本次已完成增强

### 3.1 项目管理

已完成：

1. 前端新增删除项目 API 封装。
2. 项目 store 新增 `deleteProject` 动作。
3. 项目列表页为 OWNER 增加删除按钮。
4. 删除前弹出二次确认，删除后刷新项目列表。
5. API 冒烟测试新增“OWNER 删除项目”和“删除后列表不可见”校验。

后端原本已经具备 `DELETE /api/projects/{projectId}` 和软删除逻辑，本次主要补齐前端闭环。

### 3.2 鼠标精确绘制

已完成：

1. `SceneManager` 新增 `screenToSketchPoint` 方法。
2. 鼠标坐标不再按屏幕中心粗略换算，而是通过 Three.js Raycaster 从相机发射射线，与草图 XY 平面求交。
3. 线段、矩形、圆改成按下鼠标、拖动预览、松开创建。
4. 早期默认以 5 mm 网格吸附并支持 Shift 临时精细步长，现已升级为工具栏可配置网格尺寸与捕捉开关，详见 3.25。

### 3.3 图元选择、移动与缩放

已完成：

1. 选择模式下可以点选线段、矩形、圆。
2. 选中实体会显示高亮轮廓和控制点。
3. 拖动实体主体可移动位置。
4. 拖动线段端点可修改线段端点。
5. 拖动矩形角点可修改矩形尺寸。
6. 拖动圆周或圆的半径控制点可修改半径。

### 3.4 删除、撤销和重做

已完成：

1. 工具栏新增删除按钮。
2. 支持键盘 Delete / Backspace 删除当前选中实体。
3. 删除草图实体时，会同步清理相关约束和依赖特征。
4. 工具栏撤销、重做按钮接入真实历史栈。
5. 支持 Ctrl+Z 撤销，Ctrl+Y 或 Ctrl+Shift+Z 重做。
6. 新增、更新、删除实体都会记录到快照式历史命令中。

### 3.5 属性面板参数化编辑

已完成：

1. 选中线段时可编辑起点和终点坐标。
2. 选中矩形时可编辑位置、宽度、高度。
3. 选中圆时可编辑圆心和半径。
4. 属性面板参数修改同样进入撤销栈。

### 3.6 协同操作同步

已完成：

1. 远端操作应用逻辑新增 `sketch.entity.updated`。
2. 远端操作应用逻辑新增 `sketch.entity.deleted`。
3. 远端操作应用逻辑新增 `feature.deleted`。
4. 删除图元或特征时，远端也会级联清理依赖特征。

### 3.7 视图导航与常用视角

6. 左键仍保留给绘图、选择、移动和缩放图元，避免相机控制抢占建模操作。

4. 复制动作复用 `sketch.entity.created` 协同事件，并进入撤销栈。

### 3.9 项目元数据编辑

已完成：

2. 项目 store 新增 `updateProject` 动作。

4. 新增项目编辑弹窗，可修改项目名称和描述。
5. API 冒烟测试新增“OWNER 更新项目元数据”校验。

### 3.10 三维特征参数编辑

已完成：

3. 布尔特征支持在属性面板切换 Union / Difference。
4. 特征更新会进入撤销栈。
5. 协同操作应用逻辑新增 `feature.updated`。

### 3.11 框选、多选与批量编辑

已完成：

1. `CadSelection` 新增 `sketch-entities` 多实体选择类型。
2. 视口空白区域拖拽可以框选多个草图实体。
3. `Shift` / `Ctrl` 点击实体可以增量选择或取消选择。
4. 多选状态下拖动已选实体可以批量移动整组选中对象。
5. 多选状态下复制会批量复制选中草图实体。
6. 多选状态下删除会批量删除选中草图实体，并级联清理依赖特征。
7. 场景高亮支持多个草图实体同时显示选中轮廓和控制点。
8. 属性面板会显示多选数量和可用操作。
9. 协同操作应用逻辑新增 `sketch.entities.created`、`sketch.entities.updated`、`sketch.entities.deleted`。
10. 几何冒烟脚本增加批量创建、批量更新、批量删除的远端操作验证。

### 3.12 动态尺寸反馈与视口尺寸标注

已完成：

1. 绘制线段时，视口左上角动态显示当前线段长度。
2. 绘制矩形时，视口左上角动态显示当前宽度和高度。
3. 绘制圆时，视口左上角动态显示当前半径。
4. 选中线段时，场景中显示线段长度标签。
5. 选中矩形时，场景中显示宽度和高度标签。
6. 选中圆时，场景中显示半径标签。
7. 尺寸标签采用 CanvasTexture + Sprite 渲染，并在场景重建时释放纹理资源。

### 3.13 对象吸附与定位反馈

已完成：

1. 新增 `sketchSnapping` 几何模块，统一生成草图吸附候选点。
2. 线段支持吸附端点和中点。
3. 矩形支持吸附四个角点、四条边中点和中心点。
4. 圆支持吸附圆心和四个象限点。
5. 绘制线段、矩形、圆时，鼠标靠近已有吸附点会优先以该参考点作为落点。
6. 移动或缩放实体时会排除当前正在编辑的实体，避免吸附到自身导致拖拽卡住。
7. 视口左上角显示当前吸附类型和坐标。
8. Three.js 场景中显示蓝色吸附标记和吸附类型标签。
9. 按住 `Alt` 时临时关闭对象吸附和网格吸附，用于自由落点。
10. 网格和对象捕捉后续已升级为工具栏可配置，详见 3.25。
11. 几何冒烟脚本增加圆心吸附、矩形角点吸附、排除当前编辑实体吸附的验证。

### 3.14 旋转、镜像复制与矩形阵列

已完成：

1. 新增 `sketchTransforms` 纯几何模块，封装草图实体平移、旋转、镜像和包围盒中心计算。
2. 选中单个或多个草图实体后，可以通过工具栏执行旋转，早期默认旋转 90 度，现已升级为可输入角度。
3. 旋转以当前选择集合的整体包围盒中心为基准，支持线段、矩形和圆。
4. 选中单个或多个草图实体后，可以执行“水平镜像”创建左右镜像副本。
5. 选中单个或多个草图实体后，可以执行“垂直镜像”创建上下镜像副本。
6. 选中单个或多个草图实体后，可以执行矩形阵列，早期默认 3×2 且按 30 mm 间距生成，现已升级为可输入行列数和间距。
7. 镜像和阵列不会破坏原始实体，会创建新的草图实体并选中新副本。
8. 旋转复用 `sketch.entities.updated` 操作，镜像和阵列复用 `sketch.entities.created` 操作，因此可以进入撤销/重做栈并参与协同广播。
9. 几何冒烟脚本增加选择中心计算、旋转变换、镜像变换验证。

### 3.15 固定、水平、垂直与尺寸约束闭环

已完成：

1. 新增 `sketchConstraints` 纯几何模块，封装约束应用、尺寸值读取、约束适用性判断和约束标签。
2. “固定”约束会锁定实体几何，实体仍可选中查看和修改名称，但视口拖拽与几何编辑不会改变它的位置和尺寸。
3. “水平”约束对线段生效，添加后立即将线段终点 Y 修正为起点 Y，并在后续拖拽、缩放和属性编辑中保持水平。
4. “垂直”约束对线段生效，添加后立即将线段终点 X 修正为起点 X，并在后续编辑中保持垂直。
5. “尺寸”约束会读取当前实体真实尺寸作为初始值，而不是使用固定默认值。
6. 线段尺寸约束控制长度，矩形尺寸约束依次控制宽度和高度，圆尺寸约束控制半径。
7. 右侧属性面板新增约束列表，显示固定、水平、垂直和尺寸约束。
8. 用户可以在属性面板直接修改尺寸约束值，实体几何会被约束值驱动更新。
9. 用户可以在属性面板删除约束；删除固定约束后实体恢复可拖拽。
10. 协同操作应用逻辑新增 `constraint.updated` 和 `constraint.removed`，并增强 `constraint.added`，可同步被约束驱动后的实体几何。
11. 几何冒烟脚本增加水平+尺寸约束、固定约束、尺寸初始化、远端约束新增/更新/删除验证。

### 3.16 画布级尺寸约束标注

已完成：

1. 新增 `sketchDimensionAnnotations` 纯几何模块，用于根据文档中的尺寸约束生成画布标注描述。
2. 线段长度尺寸约束会生成沿线段方向偏移的标注线和“长度 x mm”标签。
3. 矩形宽度尺寸约束会在矩形下方生成宽度标注线和“宽度 x mm”标签。
4. 矩形高度尺寸约束会在矩形右侧生成高度标注线和“高度 x mm”标签。
5. 圆半径尺寸约束会生成从圆心到半径端点的标注线和“半径 x mm”标签。
6. Three.js 场景重建时会渲染所有尺寸约束标注，因此标注不再只在实体被选中时出现。
7. 固定、水平、垂直约束会以小标签显示在实体附近，便于用户快速识别哪些实体受约束控制。
8. 几何冒烟脚本增加尺寸标注生成数量、矩形宽度标签、圆半径标签验证。

### 3.17 画布内尺寸标签直接编辑

已完成：

1. `SceneManager` 新增 `sketchPointToScreen` 方法，可将草图平面坐标投影到视口屏幕坐标。
2. `CadViewport` 基于尺寸约束标注生成透明 DOM 热点，覆盖在蓝色尺寸标签位置。
3. 鼠标悬停尺寸标签时显示轻微边框，提示该尺寸可以交互。
4. 点击尺寸标签会打开数值输入弹窗，输入新的 mm 数值后调用 `updateConstraintValue`。
5. 尺寸值修改后继续复用已有约束驱动逻辑、撤销/重做逻辑和协同广播逻辑。
6. 热点位置会随相机平移、缩放、旋转和窗口尺寸变化重新投影，避免 DOM 热点和 WebGL 标签错位。

### 3.18 同心、等半径与平行关系约束

已完成：

1. 扩展 `SketchConstraint` 数据模型，新增 `concentric`、`equalRadius`、`parallel` 三类关系约束。
2. 关系约束保留 `entityId` 作为主参考实体，并增加 `targetEntityId` 作为被驱动实体。
3. 多选两个圆后点击“同心”，第二个圆的圆心会跟随第一个圆。
4. 多选两个圆后点击“等半径”，第二个圆的半径会跟随第一个圆。
5. 多选两条线段后点击“平行”，第二条线段会保持自身长度并调整为与第一条线段同方向。
6. 文档变更后会统一重新应用关系约束，因此移动、属性编辑、尺寸修改和远端操作都会触发关系求解。
7. 删除任一相关实体时，会清理以它为主实体或目标实体的关系约束。
8. 远端操作应用逻辑新增关系约束求解，远端收到实体更新或约束新增后会自动修正目标实体。
9. 属性面板会在单选实体时显示相关关系约束，多选状态提示可用关系约束操作。
10. 几何冒烟脚本增加同心、等半径、平行，以及远端关系约束新增和源实体更新后的重新求解验证。

### 3.19 垂直关系与相切约束

已完成：

1. `SketchConstraint` 数据模型新增 `perpendicular` 和 `tangent` 两类关系约束。
2. 工具栏新增“垂直关系”按钮，用于区别单条线段的“垂直”方向约束。
3. 多选两条线段后点击“垂直关系”，第二条线段会保留自身长度，并旋转到与第一条线段垂直。
4. 多选圆和线段后点击“相切”，如果第一个实体是圆，则移动第二条线段，使线段到圆心距离等于圆半径。
5. 多选线段和圆后点击“相切”，如果第一个实体是线段，则移动第二个圆，使圆心到线段距离等于圆半径。
6. 多选两个圆后点击“相切”，第二个圆会沿两圆圆心方向移动，使两圆外切。
7. 新关系继续复用关系约束的主实体/目标实体模型、文档变更后统一求解机制和远端操作后求解机制。
8. 几何冒烟脚本增加垂直关系、圆驱动线相切、线驱动圆相切、圆圆相切验证。

### 3.20 参数化旋转与矩形阵列

已完成：

1. 工具栏“旋转90”升级为“旋转”，点击后打开参数弹窗。
2. 旋转弹窗支持输入 -360 到 360 度之间的角度，并继续围绕当前选择整体中心旋转。
3. 工具栏“3×2阵列”升级为“阵列”，点击后打开矩形阵列参数弹窗。
4. 阵列弹窗支持输入列数、行数、X 间距和 Y 间距。
5. 参数化旋转继续复用 `rotateSelection(degrees)`，进入已有历史栈和协同操作链路。
6. 参数化阵列继续复用 `arraySelection(columns, rows, spacing)`，生成的新副本会被选中并进入历史栈。
7. 几何冒烟脚本增加任意角度旋转和自定义阵列间距对应的平移验证。

### 3.21 按选中线段作为镜像轴

### 3.34 线段圆角与倒角

参考依据：

1. [Onshape 官方 Sketch Fillet 文档](https://cad.onshape.com/help/Content/Sketch/sketch_fillet.htm)说明，草图圆角可用指定半径沿一条或多条线、圆弧或样条创建圆角，并支持选择两条草图曲线后输入半径。
2. Onshape 文档还说明，当默认圆角尺寸过大时会以红色提示，并支持拖拽或数值输入调整圆角尺寸，这说明“参数化预览 + 精确输入”是成熟 CAD 的常见体验。
3. [Onshape 官方 Sketch Chamfer 文档](https://cad.onshape.com/help/Content/Sketch/sketch_chamfer.htm)说明，草图倒角可选择一个顶点或两条边，并通过输入距离值创建倒斜角。

已完成：

1. 新增 `frontend/src/cad/geometry/sketchCornerOperations.ts`，提供线段圆角和倒角的纯几何算法。
2. 线段圆角支持选中同一草图下两条未锁定线段，按输入半径修剪两条线，并创建与两条线相切的圆弧实体。
3. 线段倒角支持选中同一草图下两条未锁定线段，按输入距离修剪两条线，并创建连接两个倒角点的新线段。
4. 工具栏新增“圆角”和“倒角”按钮；参数弹窗变化时会生成临时预览，应用或取消后清空预览层。
5. 圆角/倒角操作进入统一撤销栈，并以 `sketch.entities.updated` 形式广播；远端协同应用逻辑增强为支持同一次批量更新中附带新增连接实体。
6. CAD 几何冒烟脚本新增圆角、倒角和远端 corner modification connector 的验证。

当前边界：

1. 本阶段优先支持两条线段之间的等半径圆角和等距倒角。
2. 暂未实现 Onshape 中的虚拟尖点、圆弧/样条圆角、多距离倒角和尺寸约束自动挂接，这些可作为后续参数化草图求解增强方向。

### 3.35 参数化三维基础体创建

参考依据：

1. `00附件-原始相关资料.txt` 新增要求明确提出，前端需要在 Web 中创建项目，并在项目内按参数创建球体、锥体、立方体/长方体。
2. Tinkercad、SketchUp for Web 等成熟 Web 建模工具都提供“直接放置基础体 + 参数编辑 + 拖拽定位”的入门建模路径，这比只依赖草图拉伸更适合课程演示。
3. Onshape 等机械 CAD 虽然以草图驱动特征为主，但其特征树仍保留参数化特征记录，本项目延续“参数保存到特征树、渲染层按参数重建 mesh”的设计原则。

已完成：

1. `CadDocument.features` 新增 `box`、`sphere`、`cone` 三类特征类型。
2. 工具栏新增“立方体”“球体”“锥体”按钮，分别弹出参数输入框。
3. 立方体/长方体支持长度、宽度、高度参数；球体支持半径参数；锥体支持底面半径和高度参数。
4. Three.js 几何内核新增 `BoxGeometry`、`SphereGeometry`、`ConeGeometry` 构建逻辑，并保持实体 id 与 `featureId` 的映射。
5. 左侧对象树可选中参数化基础体；右侧属性面板可精确编辑坐标、尺寸、半径和高度。
6. 中央视口支持 Raycaster 拾取基础体，选中后可拖拽移动实体位置。
7. 参数化基础体的创建、拖拽和属性修改进入撤销/重做栈，并通过 `feature.created`、`feature.updated` 参与协同同步。
8. CAD 几何冒烟脚本新增参数化基础体渲染验证和远程基础体创建/更新验证。

当前边界：

1. 锥体底面当前采用圆形底面半径，未实现多边形底面边数、椭圆底面或任意轮廓放样。
2. 基础体拖拽当前在 XY 平面内移动，Z 方向通过属性面板精确输入。
3. 视图旋转依赖 OrbitControls，实体自身旋转参数可作为后续增强。
4. 工业级布尔、倒角、抽壳和 STEP/IGES 交换仍保留给 OpenCascade.js/OCCT 后续阶段。

## 5. 后续建议增强

下一阶段建议按以下优先级继续推进：

1. 将画布尺寸编辑从弹窗升级为贴近标签的内联输入框，进一步接近成熟 CAD 的动态输入体验。

3. 增加草图坐标轴、极轴追踪和更专业的捕捉提示。

5. 将撤销/重做从本地快照升级为可协同的操作变换或版本分支策略。
6. 将 CSG 废弃警告对应的配置升级到第三方库推荐的新参数。

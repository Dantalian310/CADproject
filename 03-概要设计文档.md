# 基于云的协同机械 CAD 系统概要设计文档

## 1. 文档说明

### 1.1 文档目的

本文档在需求分析的基础上，对“基于云的协同机械 CAD 系统”进行概要设计，明确系统的总体结构、模块划分、模块职责、核心数据流、前后端代码组织方式和后续开发方向。

本文档的定位是连接需求分析和详细设计。开发人员阅读本文后，应能理解：

1. 系统整体由哪些模块组成。
2. 每个模块负责什么，不负责什么。
3. 前端、后端、数据库、WebSocket、云部署之间如何配合。
4. 后续编码时应该按什么目录和边界组织代码。
5. 哪些能力必须先做，哪些能力可以后续扩展。

### 1.2 设计依据

本文档依据以下材料：

1. 《01-可行性分析文档.md》
2. 《02-需求分析文档.md》
3. 《04附件-技术栈说明文档.md》
4. 《10-开发环境搭建与实施前准备清单.md》
5. 实验任务书与课程要求
6. FreeCAD、CrownCAD、Onshape 等 CAD 系统的参考思想

### 1.3 设计原则

本项目采用以下设计原则：

1. 主链路优先：优先完成登录、项目、建模、保存、版本、协同、部署。
2. 前端重交互：图形绘制、视角控制、基础几何生成主要在浏览器端完成。
3. 后端管数据：用户、项目、权限、版本、持久化和协同消息入口由后端负责。
4. 数据结构优先：模型数据以特征树 JSON 表达，而不是只保存最终 mesh。
5. 渐进增强：MVP 先用 Three.js 参数化几何和轻量 CSG，后续预留 OpenCascade.js/OCCT Wasm 接入点。
6. 模块清晰：前端按页面、组件、CAD 核心、协同和 API 分层；后端按业务模块分包。
7. 可演示可解释：所有核心功能都应服务于实验答辩的清晰演示。

## 2. 系统总体设计

### 2.1 系统定位

本系统是一个基于 Browser-Server 架构的轻量级协同机械 CAD 原型系统。

用户通过浏览器访问系统，在 CAD 工作台中绘制基础草图、生成简单三维模型、保存模型版本，并与其他用户实时协同编辑同一个 CAD 文档。

### 2.2 总体结构

```text
+--------------------------- 浏览器端 ----------------------------+
| Vue 3 工作台界面                                                 |
| 工具栏 | 特征树 | Three.js 视图区 | 属性面板 | 在线用户           |
|                                                                  |
| Pinia 状态层                                                     |
| 用户状态 | 项目状态 | CAD 文档状态 | 协同连接状态                  |
|                                                                  |
| CAD 核心层                                                       |
| 草图模型 | 特征树 | 几何生成 | Undo/Redo | 序列化                 |
|                                                                  |
| API 与协同客户端                                                 |
| Axios REST API | STOMP over WebSocket                           |
+----------------------------+-------------------------------------+
                             |
                  HTTP/HTTPS | WebSocket/STOMP
                             |
+----------------------------v-------------------------------------+
| Spring Boot 后端                                                  |
| Auth | User | Project | Document | Version | Collaboration       |
|                                                                  |
| Spring Security + JWT                                             |
| Spring Web MVC REST API                                           |
| Spring WebSocket + STOMP                                          |
| Spring Data JPA + Flyway                                          |
+----------------------------+-------------------------------------+
                             |
      +----------------------+------------------------+
      |                      |                        |
 PostgreSQL               Redis 可选               OBS 可选
 用户/项目/版本/JSON      在线用户/短期锁           大模型快照/导出文件
```

### 2.3 核心主流程

系统首版必须围绕以下主流程实现：

```text
注册/登录
  -> 创建项目
  -> 进入 CAD 工作台
  -> 绘制矩形或圆
  -> 拉伸生成 3D 模型
  -> 保存模型
  -> 生成版本
  -> 第二个用户加入同一文档
  -> 实时同步建模操作
  -> 查看版本历史
```

后续所有模块设计和编码任务都应围绕这条链路展开。

## 3. 模块划分

### 3.1 前端模块

前端采用 Vue 3 + TypeScript + Vite。

模块划分：

| 模块 | 主要职责 |
| --- | --- |
| 路由模块 | 登录页、项目页、工作台页的路由管理和登录守卫 |
| 用户状态模块 | token、当前用户、登录退出 |
| 项目模块 | 项目列表、项目创建、成员与基础权限分配、当前项目 |
| CAD 文档模块 | 草图、特征树、选中对象、参数、Undo/Redo |
| CAD 渲染模块 | Three.js 场景、相机、光照、网格、几何体渲染 |
| 草图交互模块 | 鼠标绘制线段、矩形、圆、椭圆、多段线、构造线，选中、删除、约束和参数编辑 |
| 特征建模模块 | 拉伸、切除、布尔并集、布尔差集、模型重建、特征树更新 |
| 属性面板模块 | 显示和编辑当前对象参数 |
| 协同模块 | STOMP 连接、订阅、发送增量操作、远程光标、在线用户、冲突提示 |
| 运行状态模块 | API 连接状态、WebSocket 状态、保存状态、在线协作者数量、最近同步错误 |
| API 模块 | Axios 封装、接口调用、错误处理 |

### 3.2 后端模块

后端采用 Spring Boot。

模块划分：

| 模块 | 主要职责 |
| --- | --- |
| common | 通用响应、异常处理、配置、工具类 |
| security | JWT、Spring Security、权限校验、密码加密 |
| auth | 注册、登录、当前用户 |
| user | 用户实体、用户查询、用户基础管理 |
| project | 项目创建、列表、详情、成员关系 |
| document | CAD 文档保存、加载、当前版本维护 |
| version | 版本快照、版本列表、历史恢复 |
| collaboration | WebSocket/STOMP、房间、操作广播、在线用户 |
| observability | 健康检查、操作日志、系统事件、结构化日志 |
| storage | 本地文件存储和 OBS 存储抽象 |

### 3.3 数据存储模块

数据存储分三类：

1. PostgreSQL：主数据源。
2. Redis：可选的协同状态和短期锁。
3. OBS：可选的大文件和模型快照对象存储。

MVP 阶段只强制使用 PostgreSQL。

### 4.2 前端核心状态

`cad.store.ts` 建议维护：

```ts
interface CadState {
  documentId: string | null
  schemaVersion: string
  sketches: Sketch[]
  features: Feature[]
  selectedObjectId: string | null
  activeTool: CadTool
  undoStack: CadCommand[]
  redoStack: CadCommand[]
  dirty: boolean
  currentVersion: number
}
```

开发要求：

1. 所有 CAD 操作必须通过 action 修改状态。
2. 不允许组件直接随意修改深层模型对象。
3. 每个会改变模型的操作都应生成 operation，便于协同广播和历史记录。

### 4.3 前端页面设计

#### 4.3.1 登录页

职责：

1. 登录。
2. 注册入口。
3. 保存 token。
4. 登录成功后跳转项目列表。

不负责：

1. CAD 数据加载。
2. WebSocket 连接。

#### 4.3.2 项目列表页

职责：

1. 展示当前用户项目。
2. 创建项目。
3. 进入 CAD 工作台。

不负责：

1. 渲染 Three.js。
2. 处理模型操作。

#### 4.3.3 CAD 工作台页

职责：

1. 加载文档。
2. 初始化 Three.js 场景。
3. 初始化协同连接。
4. 组织工具栏、特征树、属性面板、视图区。
5. 保存文档。
6. 退出时清理 WebSocket 和 Three.js 资源。

### 4.4 CAD 渲染设计

Three.js 渲染建议封装为 `SceneManager`，避免 Vue 组件直接管理大量 Three.js 对象。

`SceneManager` 职责：

1. 创建 scene、camera、renderer。
2. 创建网格、坐标轴、光源。
3. 绑定 OrbitControls。
4. 根据 CAD 文档重建 mesh。
5. 处理 resize。
6. 释放几何体、材质、事件监听。

Vue 组件职责：

1. 提供挂载 DOM。
2. 把用户操作转成 CAD command。
3. 调用 SceneManager 更新渲染。

### 4.5 几何生成设计

首版使用 `ThreeGeometryKernel`。

接口：

```ts
interface GeometryKernel {
  buildMeshes(document: CadDocument): ThreeMeshDescriptor[]
  buildExtrude(feature: ExtrudeFeature, document: CadDocument): ThreeMeshDescriptor
  buildCut(feature: CutFeature, document: CadDocument): ThreeMeshDescriptor
  buildBoolean(feature: BooleanFeature, document: CadDocument): ThreeMeshDescriptor
}
```

首版支持：

1. 矩形拉伸成长方体。
2. 圆拉伸成圆柱体。
3. Cut 切除，作为布尔差集的特化操作。
4. 基础实体的布尔并集和差集。
5. 简单草图显示。

后续扩展：

1. 更稳定的工业级 CSG/B-Rep 布尔。
2. OpenCascade.js/OCCT Wasm，并通过 `OpenCascadeJsGeometryKernel` 接入。
3. glTF/STL 导出。

### 5.2 后端分层职责

| 层 | 职责 | 编码注意 |
| --- | --- | --- |
| Controller | 接收 HTTP 请求和 STOMP 消息 | 不写复杂业务逻辑 |

### 5.3 后端核心模块职责

#### 5.3.1 Auth 模块

职责：

1. 用户注册。
2. 用户登录。
3. 密码加密。
4. JWT 生成。
5. 当前用户查询。

关键类：

1. `AuthController`
2. `AuthService`
3. `JwtTokenProvider`
4. `JwtAuthenticationFilter`

#### 5.3.2 Project 模块

职责：

1. 创建项目。
2. 查询项目列表。
3. 查询项目详情。
4. 管理项目成员，MVP 可简化。
5. 创建项目时自动创建默认 CAD 文档。

关键点：

1. 项目 owner 自动成为管理员。
2. 项目列表只返回用户有权限访问的项目。
3. 打开文档前必须校验项目权限。

#### 5.3.3 Document 模块

职责：

1. 加载当前 CAD 文档。
2. 保存 CAD 文档。
3. 更新当前版本号。
4. 校验保存权限。

关键点：

1. 保存时新增版本，不覆盖旧版本。
2. 保存操作需要事务。
3. 模型 JSON 暂存 PostgreSQL JSONB。

#### 5.3.4 Version 模块

职责：

1. 保存版本快照。
2. 查询版本列表。
3. 查询指定版本。
4. 恢复历史版本。

关键点：

1. 版本号递增。
2. 恢复历史版本时生成新版本，而不是直接覆盖当前版本。
3. 版本快照应记录创建人、时间和备注。

#### 5.3.5 Collaboration 模块

职责：

1. WebSocket/STOMP 配置。
2. 用户加入文档房间。
3. 在线用户维护。
4. 操作消息广播。
5. 光标消息广播。
6. 简单冲突提示。

关键点：

1. STOMP 连接需要鉴权。
2. 消息必须携带 documentId、userId、timestamp。
3. 服务端可以先只做转发和日志记录。
4. 复杂 CRDT 不作为 MVP。

## 8. 核心流程概要设计

### 8.1 登录流程

```text
LoginPage
  -> auth.api.login
  -> AuthController.login
  -> AuthService 校验密码
  -> 生成 JWT
  -> 前端保存 token
  -> 跳转 ProjectListPage
```

### 8.2 加载工作台流程

```text
WorkspacePage mounted
  -> document.api.getDocument(documentId)
  -> 后端校验权限
  -> 返回 current snapshot
  -> cad.store.loadDocument(snapshot)
  -> SceneManager.rebuild(document)
  -> collaboration.connect(documentId)
```

### 8.3 建模操作流程

```text
用户绘制矩形
  -> 生成 AddRectangleCommand
  -> cad.store.applyCommand(command)
  -> SceneManager.rebuild(document)
  -> collaboration.sendOperation(operation)
  -> 后端广播
  -> 其他客户端 applyRemoteOperation
```

### 8.4 保存版本流程

```text
用户点击保存
  -> cad.store.serialize()
  -> PUT /api/documents/{id}/save
  -> DocumentService 校验权限
  -> VersionService 创建新版本
  -> 更新 document.currentVersion
  -> 返回版本号
  -> 前端 dirty=false
  -> WebSocket 广播 document.saved
```

### 8.5 协同同步流程

```text
用户 A 发送 operation
  -> /app/documents/{documentId}/operations
  -> CollaborationMessageController
  -> OperationService 记录日志
  -> SimpMessagingTemplate 广播到 /topic/documents/{documentId}/operations
  -> 用户 B 收到 operation
  -> applyRemoteOperation
  -> SceneManager.rebuild
```

## 10. 编码约束

### 10.1 前端编码约束

1. Vue 组件只负责视图和事件，不直接写复杂 CAD 算法。
2. CAD 模型类型统一放在 `cad/model`。
3. Three.js 场景管理统一放在 `cad/geometry`。
4. API 调用统一放在 `api`。
5. 协同消息统一放在 `collaboration`。
6. 所有可协同的本地操作都必须转成 operation。

### 10.2 后端编码约束

1. Controller 不写复杂业务逻辑。
2. Service 负责事务和业务规则。
3. Repository 只负责数据库访问。
4. Entity 不直接返回给前端。
5. 使用 DTO 做请求和响应。
6. 使用 Flyway 管理表结构。
7. 保存版本必须使用事务。
8. WebSocket 消息必须进行基本鉴权和 documentId 校验。

## 11. 概要设计结论

本系统采用清晰的 Browser-Server 分层架构：

1. 前端 Vue 负责 CAD 工作台界面和交互。
2. Three.js 负责图形渲染和基础几何显示。
3. Pinia 负责 CAD 文档状态。
4. Spring Boot 负责业务服务、鉴权、保存、版本和协同入口。
5. PostgreSQL 负责持久化。
6. STOMP over WebSocket 负责实时协同。

后续开发应先完成主链路，再增强高级 CAD 能力。

## 1. 文档说明

本文档用于承接 `06-详细设计文档.md`、`07-数据库与数据模型设计文档.md`、`08-接口设计文档.md`、`09-原型设计与交互设计文档.md`，把编码实现阶段的前后端任务拆分、工程骨架、运行方式、Git 预留流程和本机环境情况说明清楚。

## 2. 当前本机环境检查结果

已检测到：

| 工具 | 当前状态 | 说明 |
| --- | --- | --- |
| Git | 已安装，`2.20.1.windows.1` | 当前目录原本不是 Git 仓库，已预留 Git 初始化和远程接入流程 |
| Node.js | 已安装，`v18.18.2` | 可用于 Vue + Vite |
| npm | 已安装，`10.8.1` | 前端默认使用 npm |

| Java | 已安装，`17.0.5` | Spring Boot 3 可用，当前骨架使用 Java 17 |
| Maven | 已安装，`3.9.4`，位于 `D:\Coding\Maven-3.9.4` | 后端默认使用 Maven |

环境决策：

1. 前端使用 `npm`，避免 pnpm PowerShell 执行策略问题。
2. 后端使用 Maven，不引入 Gradle。
3. Java 版本先采用本机已有 Java 17，不强制安装 Java 21。
4. Docker 暂不安装，等需要本地 PostgreSQL 容器或云端部署时再处理。
5. npm 缓存放在 `frontend/npm-cache`，Maven 缓存放在当前项目 `.cache` 目录下，减少 C 盘占用。

## 6. 前后端联调顺序

建议顺序：

1. 后端启动并通过 `/actuator/health`。
2. 前端启动并打开登录页。
3. 注册、登录、获取当前用户。
4. 创建项目并进入项目列表。
5. 打开工作台并加载默认文档。
6. 本地绘制矩形、圆并保存。
7. 创建参数化立方体、球体、锥体，并验证渲染、视图旋转、拖拽移动和属性面板尺寸编辑。
8. 查询版本列表和版本详情。
9. 两个浏览器连接同一文档，验证在线用户。
10. 验证远程光标。
11. 验证草图实体和参数化三维实体的 operation 广播。
12. 验证版本冲突和冲突提示。

## 7. Git 预留流程

当前尚未配置远程仓库地址。后续拿到地址后执行：

```powershell
git remote add origin <你的仓库地址>
git branch -M main
git push -u origin main
```

日常下拉：

```powershell
git pull --rebase origin main
```

日常上传：

```powershell
git status
git add .
git commit -m "描述本次修改"
git push
```

本项目已准备：

1. `.gitignore`。
2. `scripts/git/setup-remote.cmd`。
3. `scripts/git/pull-main.cmd`。
4. `scripts/git/push-current.cmd`。

## 8. 本地运行方式

前端：

```powershell
cd frontend
npm install
npm run dev
```

后端：

```powershell
cd backend
mvn spring-boot:run
```

脚本方式：

```powershell
scripts\dev\install-frontend.cmd
scripts\dev\frontend-dev.cmd
scripts\dev\backend-dev.cmd
```

## 9. 后续注意事项

1. 当前骨架不强制安装 Docker，避免对现有系统环境产生额外影响。
2. 如果后续需要本地 PostgreSQL，可以安装 Docker Desktop 或改用已有 PostgreSQL。
3. Docker 数据目录已预留到 `.docker-data/postgres`，便于清理。
4. npm 缓存预留到 `frontend/npm-cache`。
5. Maven 仓库预留到 `.cache/maven-repository`。
6. 云服务器部署暂不进行，等华为云 ECS 准备好后再写部署脚本。

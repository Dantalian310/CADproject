# 基于云的协同机械 CAD 系统需求分析文档

## 1. 文档说明

### 1.1 文档目的

本文档用于明确“基于云的协同机械 CAD 系统”的用户角色、业务流程、功能需求、非功能需求、数据需求、接口需求、协同需求和验收标准。

本文档将作为后续概要设计、数据库设计、接口设计、任务拆分、编码实现、测试验收和实验报告编写的依据。

### 1.2 项目范围

本系统定位为课程实验级轻量协同机械 CAD 原型系统。

系统范围包括：

1. 用户注册和登录。
2. 项目创建和项目列表。
3. 浏览器端 CAD 工作台。
4. 基础 2D 草图绘制。
5. 基础 3D 参数化建模。
6. 特征树和属性编辑。
7. 模型保存和加载。
8. 版本记录和历史查看。
9. WebSocket 多人实时协同。
10. 云端部署和演示。
11. 基础权限分配、远程光标、冲突提示、Undo/Redo。
12. 简单切除和布尔并集、差集。

系统暂不作为首版范围：

1. 完整工业级 B-Rep 几何内核。
2. 完整草图约束求解器。
3. 完整工程图模块。
4. 完整装配模块。
5. STEP/IGES/DWG 等格式完整读写。
6. FEA 仿真。
7. 商业级权限审签流程。

### 1.3 需求优先级定义

本文档使用以下优先级：

1. P0：必须实现，否则主流程不成立。
2. P1：建议实现，能明显提升项目完整性。
3. P2：可选增强，作为加分项或后续迭代。

## 2. 用户角色分析

### 2.1 角色列表

| 角色 | 说明 | 权限范围 |
| --- | --- | --- |
| 游客 | 未登录用户 | 只能访问登录、注册或演示入口 |
| 普通用户 | 已注册用户 | 创建项目、编辑自己参与的项目 |
| 项目成员 | 被加入项目的用户 | 根据权限查看或编辑项目 |
| 项目管理员 | 项目创建者或负责人 | 管理项目基本信息、成员、版本 |
| 系统管理员 | 系统维护者 | 管理系统用户和服务，MVP 可不实现独立界面 |

### 2.2 用户画像

主要用户是课程实验小组成员和答辩演示对象。

用户特点：

1. 有一定软件工程背景。
2. 不一定熟悉专业 CAD 内核。
3. 需要快速理解系统使用方式。
4. 关注系统能否演示云端协同和基础建模。

### 2.3 用户核心诉求

用户希望：

1. 不安装桌面 CAD 软件即可在浏览器中打开建模界面。
2. 可以完成简单机械零件的草图和拉伸建模。
3. 可以保存设计结果。
4. 可以与另一个用户同时编辑同一模型。
5. 可以看到在线用户和他人操作。
6. 可以查看历史版本。

## 3. 业务流程分析

### 3.1 总体业务流程

```text
用户注册/登录
  -> 创建或打开项目
  -> 进入 CAD 工作台
  -> 绘制 2D 草图
  -> 执行 3D 拉伸
  -> 修改参数
  -> 保存模型
  -> 生成版本
  -> 多人协同同步
  -> 查看历史版本
```

### 3.2 登录流程

1. 用户进入系统首页。
2. 用户输入用户名或邮箱、密码。
3. 前端调用登录接口。
4. 后端校验账号密码。
5. 后端返回 JWT。
6. 前端保存 token。
7. 用户进入项目列表页。

异常流程：

1. 用户名或密码错误：提示登录失败。
2. token 过期：跳转登录页。
3. 网络异常：提示稍后重试。

### 3.3 项目创建流程

1. 用户进入项目列表页。
2. 用户点击创建项目。
3. 用户填写项目名称和描述。
4. 前端调用项目创建接口。
5. 后端创建 project 和默认 document。
6. 前端跳转 CAD 工作台。

异常流程：

1. 项目名称为空：前端提示。
2. 创建失败：提示错误。
3. 未登录：跳转登录页。

### 3.4 CAD 建模流程

1. 用户进入 CAD 工作台。
2. 系统加载当前文档模型 JSON。
3. 用户选择矩形工具。
4. 用户在草图平面绘制矩形。
5. 系统在特征树中生成 Sketch。
6. 用户选择拉伸工具。
7. 用户输入拉伸深度。
8. 系统生成 Extrude 特征。
9. 渲染层根据特征树生成 3D Mesh。
10. 用户点击保存。
11. 系统生成版本快照。

### 3.5 协同编辑流程

1. 用户 A 和用户 B 打开同一文档。
2. 两个客户端通过 WebSocket 加入同一 document room。
3. 服务端广播在线用户列表。
4. 用户 A 绘制矩形。
5. 客户端 A 生成 operation。
6. 客户端 A 本地应用 operation。
7. 客户端 A 将 operation 发送到服务端。
8. 服务端广播给用户 B。
9. 客户端 B 应用 operation。
10. 用户 B 视图中出现相同矩形。

### 3.6 保存和版本流程

1. 用户点击保存。
2. 前端序列化当前 CAD 文档。
3. 前端调用保存接口。
4. 后端校验用户权限。
5. 后端生成 document_version 记录。
6. 后端更新 document.current_version_id。
7. 后端返回版本号。
8. 前端提示保存成功。
9. WebSocket 广播 document.saved 事件。

## 4. 功能需求

### 4.1 用户认证模块

#### FR-AUTH-001 用户注册

优先级：P0

需求描述：

系统应允许用户通过用户名、邮箱和密码注册账号。

输入：

1. 用户名。
2. 邮箱。
3. 密码。

处理：

1. 校验用户名和邮箱是否已存在。
2. 对密码进行哈希存储。
3. 创建用户记录。

输出：

1. 注册成功信息。
2. 可选返回登录 token。

验收标准：

1. 新用户可以成功注册。
2. 重复邮箱不能注册。
3. 密码不能明文保存在数据库中。

#### FR-AUTH-002 用户登录

优先级：P0

需求描述：

系统应允许用户使用邮箱或用户名加密码登录。

验收标准：

1. 正确账号密码可以登录。
2. 错误密码会提示失败。
3. 登录成功后前端获得 JWT。
4. JWT 可用于访问受保护接口。

#### FR-AUTH-003 当前用户信息

优先级：P1

需求描述：

系统应提供接口获取当前登录用户信息。

验收标准：

1. 携带有效 token 时返回用户 id、用户名、邮箱。
2. token 无效时返回未授权。

### 4.2 项目管理模块

#### FR-PROJ-001 创建项目

优先级：P0

需求描述：

用户可以创建一个 CAD 项目。

项目字段：

1. 项目名称。
2. 项目描述。
3. 创建者。
4. 创建时间。

验收标准：

1. 登录用户可以创建项目。
2. 创建项目后自动生成默认 CAD 文档。
3. 项目出现在项目列表中。

#### FR-PROJ-002 查看项目列表

优先级：P0

需求描述：

用户可以查看自己创建或参与的项目。

验收标准：

1. 用户登录后可以看到项目列表。
2. 列表显示项目名称、更新时间、创建者。
3. 点击项目可以进入 CAD 工作台。

#### FR-PROJ-003 项目成员管理

优先级：P0

需求描述：

项目管理员可以添加项目成员，并为成员分配基础权限。

验收标准：

1. 管理员可以通过用户名或邮箱添加成员。
2. 被添加成员可以在列表中看到项目。
3. 非项目成员不能打开项目。
4. 成员至少支持 OWNER 和 EDITOR 两种角色。
5. 只有 OWNER 可以添加成员或修改成员权限。

MVP 简化方案：

首版可以不做复杂邀请流程，但必须提供一种可演示的权限分配方式，例如管理员在项目成员面板中输入用户邮箱并设置角色。

### 4.3 CAD 工作台界面模块

#### FR-UI-001 工作台布局

优先级：P0

需求描述：

系统应提供符合 CAD 使用习惯的工作台界面。

布局要求：

1. 顶部工具栏。
2. 左侧特征树。
3. 中央视图区。
4. 右侧属性面板。
5. 底部状态栏。

验收标准：

1. 页面布局清晰。
2. 中央区域可以显示 3D 模型。
3. 左侧可以显示草图和特征。
4. 右侧可以显示选中对象参数。
5. 顶部可以切换建模工具。

#### FR-UI-002 工具栏

优先级：P0

需求描述：

顶部工具栏应提供常用建模操作入口。

最低工具：

1. 选择。
2. 线段。
3. 矩形。
4. 圆。
5. 椭圆。
6. 多段线。
7. 构造线。
8. 拉伸。
9. 切除。
10. 布尔并集。
11. 布尔差集。
12. 保存。
13. 撤销。
14. 重做。

验收标准：

1. 点击工具后当前工具状态改变。
2. 当前工具有视觉提示。
3. 对应视图区交互按当前工具执行。

#### FR-UI-003 状态栏

优先级：P1

需求描述：

底部状态栏显示系统状态。

显示内容：

1. 当前工具。
2. 鼠标坐标。
3. 保存状态。
4. WebSocket 连接状态。

验收标准：

1. 用户能看到当前是否已连接协同服务。
2. 保存成功或失败有明确提示。

### 4.4 2D 草图模块

#### FR-SKETCH-001 绘制线段

优先级：P0

需求描述：

用户可以在草图平面上绘制线段。

验收标准：

1. 用户选择线段工具后可以点击起点和终点。
2. 线段显示在草图平面。
3. 线段数据保存到模型 JSON。
4. 线段操作可以同步给其他用户。

#### FR-SKETCH-002 绘制矩形

优先级：P0

需求描述：

用户可以绘制矩形草图。

验收标准：

1. 用户拖拽或两点点击可以生成矩形。
2. 矩形显示宽高。
3. 矩形可以被选中。
4. 右侧属性面板可以修改宽高。
5. 矩形可以作为拉伸输入。

#### FR-SKETCH-003 绘制圆

优先级：P0

需求描述：

用户可以绘制圆形草图。

验收标准：

1. 用户指定圆心和半径生成圆。
2. 圆可以被选中。
3. 右侧属性面板可以修改半径。
4. 圆可以作为拉伸输入。

#### FR-SKETCH-004 选择和删除图元

优先级：P0

需求描述：

用户可以选择草图图元并删除。

验收标准：

1. 点击图元后出现选中高亮。
2. Delete 或删除按钮可以删除选中图元。
3. 删除操作记录到历史中。
4. 删除操作同步给其他用户。

#### FR-SKETCH-005 简化约束与尺寸驱动

优先级：P0

需求描述：

系统应支持基础尺寸驱动和简单约束，而不是完整通用约束求解器。

最低要求：

1. 矩形宽高可修改。
2. 圆半径可修改。
3. 线段端点坐标可修改。
4. 修改参数后视图实时更新。
5. 固定约束：锁定图元，禁止拖拽修改。
6. 水平约束：线段两端点保持同一 y 坐标。
7. 垂直约束：线段两端点保持同一 x 坐标。
8. 重合约束作为 P1，可先支持端点吸附表现。

验收标准：

1. 属性面板输入数值后几何图形变化。
2. 修改后的参数可以保存和加载。
3. 修改操作可以同步给其他用户。
4. 图元启用固定、水平、垂直属性后，拖拽或参数修改不会破坏约束。

#### FR-SKETCH-006 椭圆绘制

优先级：P1

需求描述：

用户可以在草图平面上绘制椭圆，用于增强 CAD 草图表达能力。

验收标准：

1. 用户指定中心点、长轴和短轴生成椭圆。
2. 椭圆可以被选中和删除。
3. 属性面板可以修改长轴、短轴和中心坐标。
4. 椭圆数据可以保存、加载和协同同步。

#### FR-SKETCH-007 多段线绘制

优先级：P1

需求描述：

用户可以连续点击绘制多段线，用于快速绘制轮廓草图。

验收标准：

1. 用户选择多段线工具后可以连续添加折线点。
2. 双击或按 Enter 完成多段线。
3. 多段线可以被选中、删除和保存。
4. 多段线操作可以协同同步。

#### FR-SKETCH-008 网格吸附与尺寸标注

优先级：P1

需求描述：

系统提供网格吸附和基础尺寸标注，使草图操作更接近 CAD 软件。

验收标准：

1. 用户可以开启或关闭网格吸附。
2. 绘制图元时端点或圆心可吸附到网格点。
3. 矩形宽高、圆半径、线段长度可以显示基础尺寸文本。
4. 尺寸文本不参与复杂工程图，只作为草图辅助信息。

### 4.5 3D 建模模块

#### FR-FEAT-001 拉伸矩形生成长方体

优先级：P0

需求描述：

用户可以将矩形草图拉伸为长方体。

输入：

1. sourceSketchId。
2. depth。

输出：

1. Extrude Feature。
2. Three.js Mesh。

验收标准：

1. 选择矩形后可以执行拉伸。
2. 输入深度后生成长方体。
3. 特征树新增拉伸节点。
4. 修改深度后模型更新。
5. 保存加载后模型仍可恢复。

#### FR-FEAT-002 拉伸圆生成圆柱体

优先级：P0

需求描述：

用户可以将圆形草图拉伸为圆柱体。

验收标准：

1. 选择圆后可以执行拉伸。
2. 输入深度后生成圆柱体。
3. 特征树新增拉伸节点。
4. 修改半径或深度后模型更新。

#### FR-FEAT-003 简单布尔操作

优先级：P0

需求描述：

系统应支持简单布尔并集和差集，满足实验最低 3D 特征要求。

验收标准：

1. 用户可以选择两个基础实体。
2. 执行并集或差集后生成新特征。
3. 特征树记录布尔操作。
4. 布尔结果可以保存、加载和协同同步。

实现边界：

首版布尔运算只要求支持基础封闭体，如长方体和圆柱体。可以使用轻量 CSG 库或简化 mesh 级布尔，不要求达到工业 B-Rep 精度。

#### FR-FEAT-004 切除 Cut

优先级：P0

需求描述：

用户可以基于草图或基础实体执行简单切除操作。

验收标准：

1. 用户可以选择一个已有实体作为被切除体。
2. 用户可以选择矩形或圆形草图作为切除轮廓。
3. 输入切除深度后生成 Cut Feature。
4. 特征树记录 Cut 节点。
5. 切除结果可以保存、加载和协同同步。

实现边界：

首版可以把 Cut 实现为布尔差集的特化操作，即 Target Solid - Cutting Solid。

### 4.6 特征树模块

#### FR-TREE-001 显示特征树

优先级：P0

需求描述：

系统应在左侧显示当前文档的草图和建模特征。

验收标准：

1. 新增草图后特征树出现 Sketch 节点。
2. 新增拉伸后特征树出现 Extrude 节点。
3. 点击节点可以选中对应对象。
4. 删除对象后特征树更新。

#### FR-TREE-002 特征参数编辑

优先级：P1

需求描述：

用户可以通过特征树选择特征，并在属性面板中修改参数。

验收标准：

1. 选择 Extrude 节点后右侧显示 depth。
2. 修改 depth 后模型重新生成。
3. 修改操作记录到历史。
4. 修改操作同步给其他用户。

### 4.7 视图区交互模块

#### FR-VIEW-001 3D 模型渲染

优先级：P0

需求描述：

系统应在中央视图区渲染 3D 模型。

验收标准：

1. 场景中有网格辅助线。
2. 场景中有坐标轴辅助。
3. 模型有基础材质和光照。
4. 生成模型后可见。

#### FR-VIEW-002 相机控制

优先级：P0

需求描述：

用户可以旋转、缩放和平移视图。

验收标准：

1. 鼠标拖动可旋转视角。
2. 滚轮可缩放。
3. 右键或中键可平移。

#### FR-VIEW-003 对象选中

优先级：P1

需求描述：

用户可以点击草图或模型对象进行选中。

验收标准：

1. 点击对象后对象高亮。
2. 属性面板显示对象信息。
3. 特征树同步选中对应节点。

### 4.8 模型保存与加载模块

#### FR-DOC-001 保存模型

优先级：P0

需求描述：

用户可以保存当前模型。

保存内容：

1. 草图实体。
2. 特征树。
3. 当前版本号。
4. 创建和修改时间。
5. 操作历史摘要。

验收标准：

1. 点击保存后后端生成版本。
2. 刷新页面后可以加载保存内容。
3. 保存失败时提示错误。

#### FR-DOC-002 加载模型

优先级：P0

需求描述：

用户进入 CAD 工作台时，系统加载当前文档数据。

验收标准：

1. 已保存模型可以恢复草图和 3D 模型。
2. 空项目显示默认空场景。
3. 加载失败时显示提示。

#### FR-DOC-003 导出 JSON

优先级：P1

需求描述：

系统可以导出当前模型 JSON，用于演示数据结构。

验收标准：

1. 用户可以下载 JSON 文件。
2. JSON 包含 sketches、features、metadata。

### 4.9 版本管理模块

#### FR-VER-001 保存版本快照

优先级：P0

需求描述：

每次用户保存模型时，系统生成一个版本快照。

验收标准：

1. 每次保存版本号递增。
2. 版本记录保存创建人和创建时间。
3. 可以通过接口查询版本列表。

#### FR-VER-002 查看版本列表

优先级：P1

需求描述：

用户可以查看当前文档的历史版本。

验收标准：

1. 版本列表显示版本号、保存人、保存时间、备注。
2. 点击版本可以加载该版本预览或恢复。

#### FR-VER-003 恢复历史版本

优先级：P1

需求描述：

用户可以将历史版本恢复为当前版本。

验收标准：

1. 恢复历史版本后当前模型变为历史快照。
2. 恢复操作本身生成新版本。
3. 恢复事件同步给在线用户。

### 4.10 Undo/Redo 模块

#### FR-HIST-001 撤销操作

优先级：P0

需求描述：

用户可以撤销最近的本地操作。

验收标准：

1. 新增图元后撤销可移除图元。
2. 修改参数后撤销可恢复旧值。
3. 撤销后视图区和特征树同步更新。

#### FR-HIST-002 重做操作

优先级：P0

需求描述：

用户可以重做被撤销的操作。

验收标准：

1. 撤销后可以重做。
2. 新操作发生后 redo 栈清空。

### 4.11 协同模块

#### FR-COL-001 加入协同房间

优先级：P0

需求描述：

用户打开文档时应自动加入对应 WebSocket 房间。

验收标准：

1. 用户进入工作台后建立 WebSocket 连接。
2. 服务端确认用户加入 document room。
3. 同一文档的用户可以互相接收事件。
4. 服务端维护当前文档房间的连接池或等价连接映射。

#### FR-COL-002 在线用户列表

优先级：P0

需求描述：

系统应显示当前文档在线用户列表。

验收标准：

1. 用户加入后列表更新。
2. 用户离开后列表更新。
3. 至少显示用户名和颜色标识。

#### FR-COL-003 操作同步

优先级：P0

需求描述：

系统应将用户的草图和特征操作同步给同一文档内其他用户。

同步操作包括：

1. 新增草图图元。
2. 修改草图参数。
3. 删除草图图元。
4. 新增拉伸特征。
5. 修改拉伸参数。
6. 保存文档事件。
7. 新增切除特征。
8. 新增布尔并集或差集特征。
9. Undo/Redo 操作产生的模型变更。

验收标准：

1. 用户 A 新增矩形，用户 B 可实时看到。
2. 用户 A 拉伸模型，用户 B 可实时看到。
3. 用户 A 修改参数，用户 B 视图更新。
4. 服务端广播的是增量 operation，而不是每次广播完整模型。
5. 同步延迟在演示环境下可接受。

#### FR-COL-004 远程光标

优先级：P0

需求描述：

系统应显示其他用户在视图区的光标位置。

验收标准：

1. 用户 A 移动鼠标时用户 B 能看到用户 A 的光标标识。
2. 光标标识包含用户名称或颜色。

#### FR-COL-005 冲突提示

优先级：P0

需求描述：

当两个用户同时修改同一对象时，系统应提示可能存在冲突。

验收标准：

1. operation 携带 baseVersion 或 objectVersion。
2. 服务端或客户端检测版本不一致。
3. 系统给出冲突提示。

MVP 简化方案：

1. 对被选中编辑的对象加临时锁。
2. 或采用后写覆盖，并提示“该对象刚被其他用户修改”。

## 5. 非功能需求

### 5.1 性能需求

#### NFR-PERF-001 页面响应

优先级：P1

要求：

1. CAD 工作台基础页面应在可接受时间内加载。
2. 常规模型旋转、缩放、平移应保持流畅。
3. 100 个以内简单草图图元不应明显卡顿。

#### NFR-PERF-002 协同延迟

优先级：P0

要求：

1. 两个用户协同操作应近实时同步。
2. 在正常网络下，操作广播延迟应适合演示。
3. WebSocket 断开后前端应显示状态。

### 5.2 可用性需求

#### NFR-USE-001 易用性

优先级：P0

要求：

1. 用户无需阅读长说明即可找到绘制和保存按钮。
2. 当前工具状态清晰。
3. 操作成功和失败有反馈。
4. 空项目应给出可开始绘制的默认视图。

### 5.3 安全需求

#### NFR-SEC-001 鉴权

优先级：P0

要求：

1. 受保护 HTTP 接口必须校验 JWT。
2. WebSocket 加入房间必须校验用户身份。
3. 用户不能访问未授权项目。

#### NFR-SEC-002 密码安全

优先级：P0

要求：

1. 密码必须哈希存储。
2. 登录失败不能泄露用户是否存在的过多细节。

### 5.4 可靠性需求

#### NFR-REL-001 数据保存可靠性

优先级：P0

要求：

1. 保存接口成功返回后，刷新页面应能恢复数据。
2. 版本快照不能覆盖旧版本。
3. 保存失败时不得破坏已有版本。

### 5.5 可维护性需求

#### NFR-MAIN-001 模块化

优先级：P0

要求：

1. 前端 CAD 状态、渲染、UI、协同、API 应分层。
2. 后端 auth、projects、documents、versions、collaboration 应分模块。
3. 前端 CAD 数据结构需要有 TypeScript 类型定义，后端请求、响应和实体需要有 Java DTO、Entity 和校验注解。

#### NFR-MAIN-002 可测试性

优先级：P1

要求：

1. 核心数据转换函数应可单元测试。
2. 后端核心接口应可用 Postman/Apifox 测试。
3. 协同流程应有手动测试脚本。

### 5.6 可部署性需求

#### NFR-DEP-001 云部署

优先级：P0

要求：

1. 系统应能部署到华为云 ECS。
2. 前端、后端、数据库服务启动步骤清晰。
3. 环境变量不应写死在代码中。
4. WebSocket 在公网访问下可用。

### 5.7 可观测性需求

#### NFR-OBS-001 运行状态与日志

优先级：P1

要求：

1. 后端应提供健康检查接口，便于本地和云端部署验证。
2. 系统应记录关键操作日志和系统事件，包括登录、保存、版本创建、WebSocket 连接、房间加入、权限拒绝和协同冲突。
3. 前端 CAD 工作台应展示 API 连接状态、WebSocket 连接状态、保存状态和在线协作者数量。
4. 协同异常或保存失败时，前端应给出明确提示，后端应保留可追踪日志。

## 9. 验收标准

### 9.1 P0 验收标准

项目必须满足：

1. 可以注册和登录。
2. 可以创建项目。
3. 可以为项目成员分配基础权限。
4. 可以进入 CAD 工作台。
5. CAD 工作台有 Ribbon 风格工具栏、左侧特征树、中间 3D 视图区、右侧属性/参数面板。
6. 可以绘制直线、矩形和圆。
7. 可以对线段启用固定、水平、垂直等基础约束。
8. 可以通过数值修改驱动矩形宽高、圆半径、线段端点等几何变化。
9. 可以拉伸生成长方体和圆柱体。
10. 可以执行简单切除 Cut。
11. 可以执行简单布尔并集和差集。
12. 可以保存模型。
13. 刷新页面后可以加载模型。
14. 每次保存生成版本记录。
15. 支持操作历史记录和 Undo/Redo。
16. 两个用户或两个浏览器窗口可以同时进入同一文档。
17. 服务端维护协同房间连接池或等价连接映射。
18. 用户 A 的增量建模操作可以同步到用户 B。
19. 可以显示在线用户列表。
20. 可以显示他人光标位置。
21. 同一对象并发修改时可以给出冲突提示。
22. 系统可以在本地稳定运行。
23. 系统可以部署到华为云 ECS 或提供可解释的部署方案。

### 9.2 P1 验收标准

建议满足：

1. 椭圆绘制。
2. 多段线绘制。
3. 构造线。
4. 网格吸附。
5. 基础尺寸标注。
6. 历史版本恢复。
7. JSON 导出。
8. 重合约束。

### 9.3 P2 验收标准

加分项：

1. Yjs 协同。
2. OpenCascade.js/OCCT Wasm 初步验证。
3. 工业级 CSG/B-Rep 布尔操作。
4. OBS 存储快照。
5. RDS 托管数据库。
6. Docker Compose 一键部署。
7. 自动化测试。
8. 工程图或装配初步能力。

## 10. 需求变更控制

为了保证按期交付，需求变更应遵循：

1. 不影响 P0 主链路。
2. 新增功能必须评估开发成本。
3. 优先修复阻塞演示的问题。
4. 高级功能不得挤占保存、加载、协同、部署时间。
5. 每次变更都应记录在任务看板或文档中。

## 12. 需求分析结论

本系统需求应围绕“轻量 CAD + 云端保存 + 多人协同”三个关键词展开。

首版最重要的不是功能数量，而是主流程闭环：

登录 -> 创建项目 -> 建模 -> 保存 -> 加载 -> 版本 -> 协同。

后续设计和实现都应服务于这条主线。所有高级功能只有在主链路稳定后才应加入。
